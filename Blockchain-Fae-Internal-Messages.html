data TXMessage a Source #

The transaction message as transmitted. This does not include the full module files, but only their "previews", containing sufficient information for the client to decide if the memory cost of requesting the module is acceptable, and to request it if so.

unsignedTXMessage :: TXMessage a -> TXMessage a Source #

Gets the "base" transaction message without validation

signTXMessage :: Serialize a => String -> PrivateKey -> TXMessage a -> Maybe (TXMessage a) Source #

Adds a single signature, overwriting one that's already there. This does not ensure that the new signature is valid, though; that is the job of unsignTXMessage.

unsignTXMessage :: Serialize a => TXMessage a -> Maybe (TXMessage a) Source #

Validates a message (all signatures present, correct, and the right identity) and returns the base message

getTXID :: Serialize a => TXMessage a -> TransactionID Source #

The transaction ID is its hash. This has to be the hash of the TXMessage structure, rather than TX, because only the former contains complete information identifying the transaction uniquely. The hash is taken of the unsigned message, without validating the signatures.

txMessageToTX :: Serialize a => Bool -> TXMessage a -> Bool -> Maybe TX Source #

Extracts the portion of the transaction that is useful for constructing the transaction call. Modules must be placed in the appropriate directory structure by the client.

validateModules :: Module -> ModuleMap -> TXMessage a -> Bool Source #

Checks the hashes of the received module files against the ones promised in the transaction. This does not validate the modules as Haskell source code.
